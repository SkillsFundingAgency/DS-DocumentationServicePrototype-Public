ILR Spec

Overview

If you receive funding directly from the Education and Skills Funding Agency (ESFA) you need to submit individualised learner records (ILRs).

ESFA uses this to calculate the funding earned by providers.

  1. Summary of changes to the specification

    Version 1 - Published January 2019.
  2. Introduction to ILR specification

    Technical guidance about collecting and submitting ILR data including:
    • Purpose of the specification
    • Intended audience
    • Background and coverage of the ILR
  3. Entities specification definitions

    Guidance for all providers covering, field definition, XML name, collections requirements and attribute codes.
  4. ILR structure

    The structure and individual field requirements for the ILR covering:
    • Learner entity
    • Learner delivery entity
    • Learner destination and progression entity
  5. Format of the ILR

    Guidance and examples of how to format and structure ILR data returns including:
    • Filename
    • Format of data
    • Header record
    • Data that is not required
    • Data types and null values
  6. Programme aims

    Rules about recording programme aims and guidance when not to.
  7. Funding models and field collection requirement tables

    Fields collected for each of the funding models when returning ILR data.

    You should send ILR data if you receive funding through one or more of the following funding models:
    • 16-19 (excluding Apprenticeships)
    • Adult skills
    • Apprenticeships (from 1 May 2017)
    • Community Learning
    • European Social Funding (ESF)
    • Other Adult
    • Other 16-19

    Rules and requirements that must be met for ILR data returns.
