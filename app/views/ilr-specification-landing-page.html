Overview

If you receive funding directly from the Education and Skills Funding Agency (ESFA) you need to submit individualised learner records (ILRs). ESFA uses this to calculate the funding earned by providers.

  1. How to submit an ILR data return To submit your individualised learner records (ILRs), you need to:
    • upload compressed files (archives or .zip files), making sure your data within these files is in a XML format and that the zip and XML files have the same file name and are not encrypted or password protected
    • make sure the data submitted has an approved employer commitment in the apprenticeship service
    • submit your ILR return before the deadline, allowing time to resolve any validation errors and warnings
    • include all learner, learning delivery and learner destination and progression records, for all learners across all funding models for the year to date
    • make one submission per returns window, unless the ILR data or data in the apprenticeship service has changed
    • make sure all your documents relating to the enrolment of the learner and the record of learning activity are accurate and conform to the eligibility rules of the appropriate type of learning
    • check that your validation reports and apprenticeship data match and correct the data causing errors
  2. Introduction to ILR What the ILR is, why it’s needed and who it’s for. Technical guidance about collecting and submitting ILR data
  3. ILR Structure Details of the structure and individual field requirements for the ILR covering:
    • Learner entity
    • Learner delivery entity
    • Learner destination and progression entity
  4. Format of the ILR Guidance and examples of how to format and structure ILR data returns including:
    • Filename
    • Format of data
    • Header record
    • Data that is not required
    • Data types and null values
  5. Programme aims Rules about recording programme aims and guidance when not to.
  6. Funding streams Check which fields need to be submitted for each funding model. Guidance for collection rules and requirements.
  7. Entity definitions Guidance for all providers covering, field definition and length, XML name, collections requirements and attribute codes.
  8. Recording valid postcodes Guidance on using the postcode field format and what to do if a postcode isn’t known
  9. ILR Learner Entry 2017 to 2018 User Guide This software is supplied free of charge to users who will submit 2017/18 ILR data to the Skills Funding Agency in the funding year but who do not have access to a data management system. This application will not be supported where data sets exceed 500 learner records.
  10. Validation rules Technical document aimed at those responsible for making data returns; data specification implementation; and MI system design (including MI managers, commercial software suppliers and own software writers)
  11. Sixth form college academy converter contracting and data guidance We have published information explaining the arrangements for sixth form colleges who convert to academy status. This will be of particular interest to those providers with apprenticeship, advanced learner loans or adult education budget contracts.
